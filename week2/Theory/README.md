# Module 2 – Theory Concepts

This section covers the theoretical concepts studied in Module 2 related to IoT systems, Arduino programming, and embedded hardware fundamentals.

## Topics Covered

### Q11 – IoT Architecture

* Four-layer IoT architecture
* Perception Layer
* Network Layer
* Processing Layer
* Application Layer
* Devices and technologies used in each layer

### Q12 – Microcontroller vs Microprocessor

Comparison between Arduino UNO and Raspberry Pi based on:

* CPU Speed
* On-chip RAM
* On-chip Flash
* Operating System
* Typical Applications
* Power Consumption

### Q13 – Arduino Pin Types

Study of different Arduino UNO pin functionalities:

* Digital Input
* Digital Output
* Analog Input
* PWM Output
* I2C Communication Pins
* SPI Communication Pins

Applications of each pin type in IoT systems.

### Q19 – analogRead() vs analogWrite()

Concepts covered:

* Analog input acquisition
* PWM signal generation
* Difference between analogRead() and analogWrite()
* Practical IoT applications

### Q20 – setup() and loop()

Arduino program execution model including:

* Purpose of `setup()`
* Purpose of `loop()`
* Effect of using long `delay()` statements
* Sensor responsiveness issues
* Non-blocking programming using `millis()`

## Learning Outcomes

By completing this module, we gained an understanding of:

* IoT architecture fundamentals
* Embedded system components
* Arduino programming structure
* Communication interfaces
* PWM concepts
* Efficient sensor data acquisition techniques
